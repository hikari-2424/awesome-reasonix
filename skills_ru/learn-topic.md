# learn-topic

研究一个技术或知识主题，输出结构化笔记。

## 流程

1. **搜索** — 用 web_search 搜 3-5 个高质量来源
2. **抓取** — 用 web_fetch 读 2-3 篇最有价值的文章
3. **整理** — 按以下结构输出：

```markdown
# [主题名称]

## 一句话总结
[≤50字]

## 核心概念
- 概念1: 解释
- 概念2: 解释

## 关键细节
[3-5个要点，每个≤3行]

## 常见误区
[如果有]

## 实践建议
[2-3条可操作的下一步]

## 参考来源
- [标题](URL)
```

## 规则
- 优先中文来源，但英文技术文档更准确时可保留
- 控制总输出在 500 字以内
- 标注信息的确定程度（确定/可能/待验证）
