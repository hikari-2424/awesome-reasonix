# git-helper

Помощь в ежедневных операциях с Git.

## Операции

### commit
1. Запустите `git diff --staged` для просмотра изменений
2. Сгенерируйте сообщение в стиле Conventional Commits
3. Покажите предложенное сообщение, подтвердите перед коммитом

### pr
1. Запустите `git log main..HEAD --oneline`
2. Сгенерируйте шаблон описания PR

### log
1. Запустите `git log --oneline -n <количество>`
2. Сгруппируйте по типу, отметьте подозрительные коммиты

### conflict
1. Покажите конфликтующие файлы
2. Проанализируйте причины конфликта
3. Предложите стратегию разрешения

## Правила
- Никогда не делайте push автоматически
- Не изменяйте директорию .git/
- Первая строка сообщения коммита ≤ 72 символа
