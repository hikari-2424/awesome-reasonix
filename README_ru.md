## Что это

Reasonix расширяется с помощью **Skills** — markdown-файлов в `.reasonix/skills/`. Здесь сообщество собирает, оценивает и делится ими.

**Проблема**: каждый пишет свой git-помощник, свою проверку кода, свой файловый органайзер. **Решение**: напишите один раз, поделитесь здесь, пусть другие оценят и улучшат.

## Установка

```bash
curl -o .reasonix/skills/file-organizer.md https://raw.githubusercontent.com/hikari-2424/awesome-reasonix/main/skills/file-organizer.md
```

Запустите Reasonix и введите `/skill file-organizer`. Готово.

Два примера:

| Skill | Делает |
|---|---|
| [file-organizer](skills/file-organizer.md) | Классификация файлов, дубликаты, дерево каталогов. |
| [git-helper](skills/git-helper.md) | Conventional Commits, описания PR, анализ истории. |

## MCP-серверы

Совместимые с Reasonix [MCP](https://modelcontextprotocol.io/)-серверы:

- [filesystem](https://github.com/modelcontextprotocol/servers/tree/main/src/filesystem) — доступ к файловой системе.
- [github](https://github.com/modelcontextprotocol/servers/tree/main/src/github) — репозитории, PR, issues.
- [puppeteer](https://github.com/modelcontextprotocol/servers/tree/main/src/puppeteer) — headless-браузер.
- [memory](https://github.com/modelcontextprotocol/servers/tree/main/src/memory) — граф знаний.

## Участие

Откройте [issue для отправки](https://github.com/hikari-2424/awesome-reasonix/issues/new?template=skill-submit.yml) или пришлите PR.

Автоматические проверки формата и безопасности. Прошёл → опубликовано сразу. Не прошёл → комментарий с причиной. Ручная модерация не требуется.

Навыки с низким рейтингом или жалобами скрываются автоматически.

Руководство: [Как создать навык](guides/how-to-create-a-skill.md) · [CONTRIBUTING.md](CONTRIBUTING.md)
